# Documentation Overview

## Purpose
- Provide easy-to-read references for supervisors, PMs, and engineers.
- Capture business scope, technical expectations, security posture, and metadata governance.

## What's inside
- `BRD.md` — business requirements, scope, risks, and success metrics.
- `FUNCTIONAL_REQUIREMENTS.md` — feature checklist organised by user role.
- `NON_FUNCTIONAL_REQUIREMENTS.md` — performance, reliability, security, and operations targets.
- `API_CONTRACTS.md` — REST endpoint catalogue with plain-language descriptions.
- `DATA_MODEL_REFERENCE.md` — canonical datasets, logical field dictionary, and mapping templates.
- `METADATA_LAYER_SPEC.md` — governance metadata design for auditability and compliance.
- `ARCHITECTURE_FILE_STRUCTURE.md` — guide to the codebase structure and module responsibilities.
- `RBAC_MODEL.md` — role definitions and access scopes (populate during security design).
- `SECURITY_GUIDE.md` — security posture checklist, currently awaiting detailed content.
- `STAKEHOLDER_DECISIONS.md` — log of open questions needing business or compliance input.

Additional documents will be added here as they are drafted; update the list when new files appear.

## How to use
- Start with the BRD to align on scope.
- Refer to other documents when planning or reviewing work packages.
- Share with stakeholders who need clarity without diving into source code.
